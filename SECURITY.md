# Security Policy

## Supported Versions

Security updates are provided for the latest release on the default branch.
If you are using an older release, please upgrade before reporting issues.

## Reporting a Vulnerability

Please report security issues privately. Do **not** open a public issue.

Preferred options:

1. GitHub Security Advisories (recommended).
2. Email: security@workbench.dev

Include as much detail as possible:

- A description of the vulnerability and impact.
- Steps to reproduce or a proof of concept.
- Any relevant logs or screenshots.

We aim to acknowledge reports within 5 business days and will coordinate a
resolution timeline with you.
